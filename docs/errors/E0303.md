error[E0303]: await outside of async function

This error occurs when you use the `await` keyword outside of an `async`
function or block.

Example of erroneous code:
    fn main() {
        let result = fetch_data().await;  // error: await outside of async
    }

    fn process() {
        let data = get_async_data().await;  // error: await outside of async
    }

Common causes:
- Calling async functions from synchronous code without proper handling
- Forgetting to mark a function as async
- Misunderstanding the async/await model

To fix this, use async functions or blocks:

Correct examples:
    async fn fetch_and_process() {
        let result = fetch_data().await;  // ok: inside async fn
        process(result);
    }

    async fn main() {
        let data = get_async_data().await;  // ok: main is async
    }

If you need to call async code from a sync function, use a runtime:
    fn main() {
        // Start the async runtime
        runtime::block_on(async {
            let result = fetch_data().await;
            println(result);
        });
    }

Or spawn the async task:
    fn trigger_fetch() {
        runtime::spawn(async {
            let data = fetch_data().await;
            handle_data(data);
        });
    }

Understanding async/await:
- `async fn` defines an asynchronous function
- `.await` suspends execution until the async operation completes
- `await` can only be used inside `async` contexts
- Async functions return Future<T> instead of T
