error[E0201]: wrong number of arguments

This error occurs when a function is called with a different number of
arguments than it was defined to accept.

Example of erroneous code:
    fn greet(name: str) {
        println("Hello, " + name);
    }

    fn main() {
        greet();              // error: takes 1 argument but 0 were supplied
        greet("Alice", 30);   // error: takes 1 argument but 2 were supplied
    }

    fn add(a: i64, b: i64) -> i64 {
        a + b
    }

    fn test() {
        add(1);          // error: takes 2 arguments but 1 was supplied
        add(1, 2, 3);    // error: takes 2 arguments but 3 were supplied
    }

Common causes:
- Forgetting to pass required arguments
- Passing extra arguments
- Confusing function signatures
- Copy-paste errors

To fix this, match the number of arguments to the function signature:

Correct examples:
    fn greet(name: str) {
        println("Hello, " + name);
    }

    fn main() {
        greet("Alice");  // ok: 1 argument as expected
    }

    fn add(a: i64, b: i64) -> i64 {
        a + b
    }

    fn test() {
        add(1, 2);  // ok: 2 arguments as expected
    }

If you need optional parameters, consider:
1. Using default values (if supported)
2. Creating overloaded functions with different arities
3. Using Option<T> for optional parameters

See also:
- E0101: cannot find function in this scope
- E0200: mismatched types
- E0202: wrong number of type arguments
