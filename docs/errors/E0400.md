error[E0400]: file not found

This error occurs when the compiler cannot find a source file that was
specified or imported.

Example of erroneous code:
    // Trying to compile a non-existent file
    $ sxc build missing_file.sx
    error[E0400]: file not found: missing_file.sx

    // Importing a module that doesn't exist
    use nonexistent_module;  // error: file not found

Common causes:
- Typo in filename or path
- File was moved, renamed, or deleted
- Incorrect working directory
- Missing file extension
- Module file doesn't follow naming convention

To fix this:

1. Check the filename spelling:
    $ sxc build hello.sx  # make sure hello.sx exists

2. Verify the file exists:
    $ ls -la *.sx
    hello.sx
    main.sx

3. Use the correct path:
    $ sxc build src/main.sx  # include directory if needed

4. For module imports, ensure the file exists:
    // If you have: use math::utils;
    // You need: math/utils.sx or math.sx with utils module

    // File structure should be:
    // src/
    //   main.sx
    //   math/
    //     utils.sx    <- for `use math::utils`
    //   math.sx       <- or module defined here

5. Check file permissions:
    $ chmod +r myfile.sx  # ensure file is readable

Common file organization patterns:
    project/
      src/
        main.sx         <- entry point
        lib.sx          <- library root
        utils.sx        <- use utils;
        math/
          mod.sx        <- use math;
          vector.sx     <- use math::vector;
