error[E0203]: trait not implemented

This error occurs when you try to use functionality that requires a trait
implementation, but the type doesn't implement that trait.

Example of erroneous code:
    trait Display {
        fn to_string(self) -> str;
    }

    struct Point {
        x: i64,
        y: i64,
    }

    fn print_it<T: Display>(item: T) {
        println(item.to_string());
    }

    fn main() {
        let p = Point { x: 1, y: 2 };
        print_it(p);  // error: `Point` does not implement trait `Display`
    }

Common causes:
- Using a generic function with trait bounds on a type that lacks the impl
- Calling trait methods on a type without the implementation
- Forgetting to add #[derive(...)] attributes
- Trait is implemented but not in scope

To fix this, implement the trait for your type:

Correct example:
    trait Display {
        fn to_string(self) -> str;
    }

    struct Point {
        x: i64,
        y: i64,
    }

    impl Display for Point {
        fn to_string(self) -> str {
            format("({}, {})", self.x, self.y)
        }
    }

    fn print_it<T: Display>(item: T) {
        println(item.to_string());
    }

    fn main() {
        let p = Point { x: 1, y: 2 };
        print_it(p);  // ok: Point implements Display
    }

For common traits, you might be able to derive them:
    @derive(Debug, Clone, Eq)
    struct Point {
        x: i64,
        y: i64,
    }

See also:
- E0108: cannot find trait in this scope
- E0107: no method found
