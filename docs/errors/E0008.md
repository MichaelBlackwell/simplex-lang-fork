error[E0008]: expected pattern

This error occurs when the parser expected a pattern (for matching or binding)
but found something else.

Example of erroneous code:
    fn main() {
        let = 42;             // error: expected pattern before `=`

        match x {
            => println("hi"), // error: expected pattern before `=>`
        }
    }

Common causes:
- Missing variable name in let binding
- Missing pattern in match arm
- Invalid pattern syntax in function parameters
- Missing identifier in for loop

To fix this, provide a valid pattern:

Correct examples:
    fn main() {
        let x = 42;           // ok: identifier pattern
        let (a, b) = pair;    // ok: tuple destructuring pattern

        match x {
            0 => println("zero"),
            n => println("other"),  // ok: binding pattern
        }
    }

Valid patterns include:
- Identifiers: x, my_var
- Literals: 42, "hello"
- Wildcards: _
- Tuple patterns: (a, b)
- Struct patterns: Point { x, y }
- Enum variant patterns: Some(x), None

See also:
- E0003: expected token
- E0007: expected expression
