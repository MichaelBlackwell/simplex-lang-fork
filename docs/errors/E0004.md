error[E0004]: invalid literal

This error occurs when a literal value (number, string, etc.) is malformed
and cannot be parsed correctly.

Example of erroneous code:
    fn main() {
        let x = 12.34.56;     // error: invalid float literal
        let y = 0xGHI;        // error: invalid hex literal
        let z = "hello;       // error: unterminated string literal
    }

Common causes:
- Multiple decimal points in a float literal
- Invalid digits for the number base (e.g., 'G' in hex)
- Unterminated string or character literals
- Invalid escape sequences in strings
- Overflow in numeric literals

To fix this:
1. Check number literals for valid format
2. Ensure strings are properly quoted and terminated
3. Use valid escape sequences (\\n, \\t, \\\\, \\", etc.)
4. Use appropriate size for numeric values

Correct examples:
    fn main() {
        let x = 12.34;        // ok: valid float
        let y = 0xABC;        // ok: valid hex
        let z = "hello";      // ok: properly terminated
    }

Valid numeric formats:
- Decimal: 42, 1_000_000
- Hex: 0xFF, 0xDEAD_BEEF
- Binary: 0b1010
- Float: 3.14, 1.0e10
