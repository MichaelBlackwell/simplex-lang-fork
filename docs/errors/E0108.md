error[E0108]: cannot find trait in this scope

This error occurs when you reference a trait that hasn't been defined
or imported into the current scope.

Example of erroneous code:
    impl Display for Point {  // error: cannot find trait `Display`
        fn fmt(self) -> str {
            // ...
        }
    }

    fn print<T: Printable>(item: T) {  // error: cannot find trait `Printable`
        item.print();
    }

Common causes:
- Using a trait that hasn't been defined
- Typo in trait name
- Trait defined in another module without import
- Confusing trait with struct or type

To fix this, define the trait or import it:

Correct examples:
    trait Display {
        fn fmt(self) -> str;
    }

    struct Point {
        x: i64,
        y: i64,
    }

    impl Display for Point {
        fn fmt(self) -> str {
            format("({}, {})", self.x, self.y)
        }
    }

With module imports:
    use std::fmt::Display;

    impl Display for MyType {
        fn fmt(self) -> str {
            // ...
        }
    }

For trait bounds on generics:
    trait Comparable {
        fn compare(self, other: Self) -> i64;
    }

    fn max<T: Comparable>(a: T, b: T) -> T {
        if a.compare(b) > 0 { a } else { b }
    }

See also:
- E0102: cannot find type in this scope
- E0203: trait not implemented
