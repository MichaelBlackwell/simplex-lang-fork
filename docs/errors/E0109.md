error[E0109]: duplicate definition

This error occurs when you define something (variable, function, struct, etc.)
with a name that's already in use in the same scope.

Example of erroneous code:
    fn main() {
        let x = 1;
        let x = 2;  // error: duplicate definition of `x`
    }

    fn add(a: i64, b: i64) -> i64 { a + b }
    fn add(a: i64, b: i64) -> i64 { a + b }  // error: duplicate function `add`

    struct Point { x: i64, y: i64 }
    struct Point { a: f64, b: f64 }  // error: duplicate struct `Point`

Common causes:
- Accidentally defining the same variable twice
- Copy-paste errors with functions
- Multiple modules defining the same name without namespacing
- Forgetting that a name was already used

To fix this:
1. Use a different name for the second definition
2. Remove the duplicate definition
3. Use modules/namespaces to separate definitions

Correct examples:
    fn main() {
        let x = 1;
        let y = 2;  // ok: different name
    }

    // Or use shadowing intentionally in a new scope:
    fn main() {
        let x = 1;
        {
            let x = 2;  // ok: shadows outer x in this scope
            println(x); // prints 2
        }
        println(x);     // prints 1
    }

For functions with different signatures, consider:
    fn add_i64(a: i64, b: i64) -> i64 { a + b }
    fn add_f64(a: f64, b: f64) -> f64 { a + b }

Or use generics:
    fn add<T: Add>(a: T, b: T) -> T { a + b }

See also:
- E0100: cannot find value in this scope
- E0101: cannot find function in this scope
