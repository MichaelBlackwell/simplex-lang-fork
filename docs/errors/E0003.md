error[E0003]: expected token

This error occurs when the parser expected a specific token but found
something else.

Example of erroneous code:
    fn main() {
        let x 42;  // error: expected `=`, found `42`
    }

    fn add(a: i64 b: i64) -> i64 {  // error: expected `,`, found `b`
        a + b
    }

Common causes:
- Missing assignment operator `=` in let bindings
- Missing commas between function parameters
- Missing colons in type annotations
- Missing semicolons at end of statements
- Missing `->` in function return types

To fix this, add the expected token:

Correct examples:
    fn main() {
        let x = 42;  // ok: added `=`
    }

    fn add(a: i64, b: i64) -> i64 {  // ok: added `,`
        a + b
    }

See also:
- E0001: unexpected token
- E0006: expected type
