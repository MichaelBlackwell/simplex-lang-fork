error[E0105]: cannot find enum in this scope

This error occurs when you try to use an enum type that hasn't been
defined or imported into the current scope.

Example of erroneous code:
    fn main() {
        let status = Status::Active;  // error: cannot find enum `Status`
    }

    fn process(result: Result<i64>) {  // error if Result not in scope
    }

Common causes:
- Using an enum that hasn't been defined
- Typo in enum name
- Enum defined in another module without import
- Confusing enum with struct

To fix this, define the enum or import it:

Correct examples:
    enum Status {
        Active,
        Inactive,
        Pending,
    }

    fn main() {
        let status = Status::Active;  // ok
    }

With module imports:
    use http::StatusCode;

    fn handle(code: StatusCode) {
        match code {
            StatusCode::Ok => println("success"),
            StatusCode::NotFound => println("not found"),
            _ => println("other"),
        }
    }

Common built-in enums (if available in prelude):
    Option<T> - Some(value) or None
    Result<T, E> - Ok(value) or Err(error)

Example with Option:
    fn find(arr: [i64], target: i64) -> Option<i64> {
        // ...
        Option::Some(index)  // or Option::None
    }

See also:
- E0102: cannot find type in this scope
- E0106: no variant on enum
