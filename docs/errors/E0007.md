error[E0007]: expected expression

This error occurs when the parser expected an expression but found
something else (like a keyword, delimiter, or end of statement).

Example of erroneous code:
    fn main() {
        let x = ;             // error: expected expression
        let y = if { };       // error: expected expression after `if`
        return;               // error (if return type expected): expected expression
    }

Common causes:
- Empty right-hand side of assignment
- Missing condition in if/while/for statements
- Missing expression after operators
- Missing return value when function expects one

To fix this, provide a valid expression:

Correct examples:
    fn main() {
        let x = 42;           // ok: expression provided
        let y = if true { 1 } else { 2 };  // ok: condition provided
    }

    fn get_value() -> i64 {
        return 42;            // ok: return expression provided
    }

Valid expressions include:
- Literals: 42, "hello", true
- Variables: x, my_var
- Function calls: foo()
- Binary operations: a + b
- Blocks: { expr }
- Conditionals: if cond { a } else { b }

See also:
- E0001: unexpected token
- E0003: expected token
