error[E0200]: mismatched types

This error occurs when the compiler expected one type but found another.

Example of erroneous code:
    fn main() {
        let x: i64 = "hello";  // error: expected `i64`, found `str`
    }

    fn add(a: i64, b: i64) -> i64 {
        a + b
    }

    fn test() {
        let result = add(1, "two");  // error: expected `i64`, found `str`
    }

    fn get_number() -> i64 {
        "not a number"  // error: expected `i64`, found `str`
    }

Common causes:
- Assigning a value of the wrong type to a variable
- Passing arguments of the wrong type to a function
- Returning the wrong type from a function
- Mixing incompatible types in expressions

To fix this, ensure types match:

Correct examples:
    fn main() {
        let x: i64 = 42;  // ok: types match
        let s: str = "hello";  // ok: types match
    }

    fn add(a: i64, b: i64) -> i64 {
        a + b
    }

    fn test() {
        let result = add(1, 2);  // ok: both arguments are i64
    }

If you need to convert between types, use explicit casting:
    fn main() {
        let i: i64 = 42;
        let f: f64 = i as f64;  // ok: explicit cast
    }

For string to number conversion, use parsing functions:
    fn main() {
        let s = "42";
        let n = parse_i64(s);  // ok: parse string to number
    }

See also:
- E0204: invalid cast
- E0205: cannot infer type
