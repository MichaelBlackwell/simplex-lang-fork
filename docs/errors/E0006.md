error[E0006]: expected type

This error occurs when the parser expected a type annotation but found
something else.

Example of erroneous code:
    fn main() {
        let x: = 42;          // error: expected type after `:`
    }

    fn add(a: , b: i64) -> i64 {  // error: expected type for parameter
        a + b
    }

    struct Point {
        x:,                   // error: expected type for field
        y: i64,
    }

Common causes:
- Missing type after `:` in variable declarations
- Missing type in function parameter definitions
- Missing type in struct field definitions
- Incomplete generic type arguments

To fix this, provide a valid type:
- Primitive types: i64, f64, bool, str
- User-defined types: struct names, enum names
- Generic types: Vec<T>, Option<T>
- Function types: fn(i64) -> i64

Correct examples:
    fn main() {
        let x: i64 = 42;      // ok: type specified
    }

    fn add(a: i64, b: i64) -> i64 {  // ok: types specified
        a + b
    }

    struct Point {
        x: f64,               // ok: type specified
        y: f64,
    }

See also:
- E0003: expected token
- E0102: cannot find type in this scope
